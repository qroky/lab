# INFO-028 — Identity-защита самоизменения

## Слова CEO (verbatim)

> Identity-защита самоизменения (от CEO): (1) мета-инвариант в жёсткое
> ядро: правила об изменении правил (границы micro/touch, состав
> жёсткого ядра, обязательность human-гейта на конституционных
> касаниях) меняются только полным touch-циклом с подписью CEO —
> никогда micro-tier; (2) аудит соответствия замыслу в release-чеклист:
> при freeze L-tier сверяет действующую конституцию+скилл с MANIFEST и
> founding-намерениями — вердикт «куда сдрейфовали» с классификацией
> (законная эволюция с решениями-основаниями / неосознанный дрейф →
> человеку); прецедент-канон: потеря перспективной оси, пойманная
> Handover1; (3) maturity для поведенческих знаний: прецеденты и знания,
> влияющие на решения, несут уровень зрелости (draft после 1 случая —
> цитируется с оговоркой; validated после N подтверждений); (4)
> зафиксировать в UPDATES.md для пользователей: их инстанс защищён так
> же — ядро не правится локальными изменениями, всё восстановимо по
> git-истории.

## Interpretation

1. **Мета-инвариант (п.1)** — рекурсивная защита: сегодняшний поток
   micro-tier правок легален, потому что он не трогает правила об
   изменении правил; сами эти правила (границы micro/touch, состав
   жёсткого ядра INFO-027, human-гейт на конституционных касаниях)
   меняются ТОЛЬКО полным touch-циклом с подписью CEO. По собственной
   логике инвариант не может быть внесён в ядро micro-tier'ом —
   поэтому: практика действует немедленно (агент обязан отклонять
   micro-tier заявки на такие изменения), кодификация — кандидат 19
   touch №2 под подпись CEO. Консистенси-скан (INFO-027) получает
   четвёртый элемент жёсткого ядра: rules-about-rules.
2. **Аудит соответствия замыслу (п.2)** — пятый постоянный шаг freeze:
   L-tier кладёт рядом действующую конституцию+скилл и
   MANIFEST/founding-намерения, вердикт «куда сдрейфовали» с закрытой
   классификацией: (а) законная эволюция — у каждого сдвига есть
   решение-основание в decisions/; (б) неосознанный дрейф — основания
   нет → человеку. Канон: потеря перспективной оси, пойманная
   Handover1 (система месяц жила без мульти-линз, никто не решал этого).
3. **Maturity поведенческих знаний (п.3)** — расширение INFO-014 с
   ролей на знания: every knowledge файл, влияющий на решения, несёт
   `Maturity:` (draft — 1 случай, цитируется с оговоркой «единичный
   прецедент»; validated — N подтверждений, N объявляется в файле).
   Разметка существующих файлов — same-day.
4. **Защита пользователя (п.4)** — строка в требования docs/UPDATES.md
   (ATOM-120 H8): инстанс пользователя защищён тем же принципом — ядро
   (vendored framework) не правится локальными изменениями втихую,
   любое состояние восстановимо по git-истории; его decisions/ — его
   журнал изменений себя.
