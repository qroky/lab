# INFO-018 — Правило соразмерности спавна

## Слова CEO (verbatim)

> Правило соразмерности спавна (от CEO, micro-tier сейчас +
> touch-кандидат): решение о порождении субагента/роли взвешивает три
> величины в масштабе ВСЕГО проекта, не текущей задачи: сложность
> (требует ли отдельного взгляда), полная цена (включая ~50k
> фикс-оверхед субагента + verify), ценность для миссии проекта (что
> изменится в конечном результате — одной строкой). Механика: (1) O5.1
> justification получает обязательное поле «ценность для миссии» и
> «почему не хватило ступени ниже» по лестнице: inline < E1-вопрос
> человеку < дешёвая линза S/M < субагент с ролью; (2) карта перспектив
> вправе вычеркнуть релевантную-но-неокупаемую линзу с записью
> (NOT-DOING уровня спавна), пример-канон: линза почвенного анализа при
> миссии «минимальный уход» → заменяется одним E1-вопросом; (3) здоровье
> правила меряется subtree-весом класса деревьев при неизменном качестве
> (verify-возвраты, гейт-правки). Связка: это вторая половина принципа
> Роуэна — «не наряжай в экспедицию то, что решается одним вопросом».

## Interpretation

1. Масштаб взвешивания — миссия проекта (mission.md, INFO-011), не
   локальная задача: линза может быть релевантна вопросу и при этом
   неокупаема для миссии — тогда она вычёркивается с записью.
2. O5.1-запись при каждом спавне дополняется двумя обязательными
   полями: `mission_value:` (одна строка — что изменится в конечном
   результате) и `why_not_lower:` (почему не хватило ступени ниже по
   лестнице inline < E1 < S/M-линза < субагент с ролью).
3. Лестница и Роуэн — две половины одной дисциплины, направленные в
   разные стороны: Роуэн (INFO-012) запрещает спрашивать человека о
   том, что берётся разведкой в конверте; соразмерность запрещает
   снаряжать разведэкспедицию там, где хватает одного вопроса. E1 в
   лестнице стоит ниже линзы по ЦЕНЕ; легитимность самого E1 по-прежнему
   проверяется Роуэном (rowan-test, INFO-013): вопрос, чей ответ есть в
   артефактах, не легитимен ни на какой ступени.
4. NOT-DOING уровня спавна: карта перспектив записывает вычеркнутую
   линзу и её замену (обычно один E1-вопрос) — вычёркивание видно
   человеку и verify, не молчаливо.
5. Метрика здоровья: subtree-вес деревьев одного класса должен падать
   при неизменном качестве (verify-возвраты, объём гейт-правок) —
   считается рядом с L/M/S-долями и rowan-fail (O4.3-семейство).
6. Конституционный дом — touch №2, кандидат 14 (O5.1 поля + право
   карты перспектив на вычерк + метрика).
