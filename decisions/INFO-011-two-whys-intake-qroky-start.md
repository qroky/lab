# INFO-011 — Два «зачем» при обустройстве + команда «qroky start»

## Слова CEO (суть, подтверждено «да, всё верно» после пересказа)

Контекст: Кроки, обустроенный в miaPOS, сфокусировался на текущем списке
задач, тогда как у владельца проекта горизонт много шире; владелец ждёт
анализа и подсветки слепых зон.

> …при обустройстве как раз попытаться выйти на уровень выше, чтобы
> ответить на второй вопрос «почему»: «для чего мы это делаем?» — а когда
> есть ответ, ещё раз: «а вот это — для чего?» […] второй потолок позволяет
> найти первоначальный смысл того, что делается. Если при обустройстве
> Кроки будет делать этот intake — это by default расширит горизонт
> понимания проекта, в который он попадает. Иногда он попадает в новый
> проект, иногда в существующий. Нужен не горизонт текущего списка задач,
> а горизонт уровня выше или двух уровней выше.

Дополнение (verbatim): «добавь ещё команду обустройства как "qroky Start",
потому что обустройся — так себе слово, не всегда для всех понятно и
работает. Qroky Start будет работать на всех языках и понятно для всех.»

## Interpretation
1. Лестница из двух «зачем» — обязательная часть ЛЮБОГО обустройства:
   уровень 1 — локальная цель текущей работы; уровень 2 — первоначальный
   смысл. Два потолка, дальше не лезть (правило остановки).
2. Существующий проект: система сначала РЕКОНСТРУИРУЕТ оба уровня из
   артефактов и приносит человеку на подтверждение; расхождение
   реконструкции с ответом человека — находка первого класса (дрейф
   проекта). Новый проект: два «зачем» задаются человеку напрямую.
3. Подтверждённые «зачем» дословно → `<project>/qroky/mission.md` ДО
   принятия первой задачи; mission.md — точка отсчёта для карт покрытия,
   границ этапов, аудитов слепых зон.
4. `qroky start` ≡ `кроки, обустройся` — языконезависимая команда
   обустройства, четвёртая форма жеста.
5. Deletion test пройден: без этого шага система наследует горизонт
   бэклога — дефект, наблюдённый в miaPOS.
