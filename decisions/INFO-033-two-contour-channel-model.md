## Формулировка CEO (дословно)

> Финальная модель каналов Telegram-головы (решение CEO, заменяет обе
> предыдущие правки): два контура. Контур диалога — событийный, мгновенный,
> двусторонний: сообщение пользователя → ответ сразу (listener будит
> обработку; на сложное — честное «принял, результат к N» без потерь и
> молчания); событие системы (гейт/E1/результат/просроченная зависимость) →
> пользователю сразу, в момент возникновения. Контур дайджеста — строго по
> расписанию (время из профиля, дефолт утро): сводка «сделано / в работе /
> ждёт тебя сегодня / расход»; уже просигналенное событиями не дублируется
> алармом — живёт строкой статуса. Оба контура настраиваются в профиле
> независимо (время дайджеста; часы тишины для событийного — ночь).
> Hard-критерии: сообщение пользователя никогда не остаётся без ответа
> >1 мин (хотя бы подтверждение); событие-блокер доставляется в момент
> возникновения; дайджест приходит во время из профиля ±5 мин ежедневно.

## Интерпретация (закрытые решения)

1. **Контур диалога** — событийный, мгновенный, двусторонний:
   - входящее сообщение пользователя получает ответ сразу; listener БУДИТ
     обработку (сам шлёт мгновенное подтверждение, затем поднимает
     обработчик); на сложное — честное «принял, результат к N», и N
     держится: без потерь и без молчания;
   - события системы — **гейт / E1 / результат / просроченная
     зависимость** — уходят пользователю в момент возникновения (шлёт та
     сторона, где событие родилось: сессия или heartbeat, через общий
     send-helper).
2. **Контур дайджеста** — строго по расписанию: время из профиля (дефолт —
   утро); состав: «сделано / в работе / ждёт тебя сегодня / расход».
   Событие, уже просигналенное контуром диалога, в дайджесте НЕ дублируется
   алармом — живёт строкой статуса.
3. **Профиль настраивает контуры независимо:** время дайджеста; часы тишины
   для событийного контура (ночь) — события в тишине копятся и доставляются
   по её окончании (блокеры — первыми).
4. **Hard-критерии (дословно в DoD):** (а) сообщение пользователя никогда
   не остаётся без ответа >1 мин — хотя бы подтверждение; (б)
   событие-блокер доставляется в момент возникновения (вне часов тишины);
   (в) дайджест приходит во время из профиля ±5 мин ежедневно.
5. **Следствие для физики (поверх INFO-032):** «подхват сессией или
   heartbeat'ом» усиливается до «listener будит обработку» — listener
   остаётся скриптом (не LLM-агентом, без входящих портов), но получив
   входящее: мгновенно подтверждает и поднимает обработчик; polling-такт
   ужимается так, чтобы держать критерий 1 минуты. Файловая шина
   decisions/inbox и сценарий «закрытая сессия → не потеряно» остаются в
   силе (страховка на случай, когда обработчик поднять нельзя).
6. **Лента NARRATIVE (интерпретация, не дословное):** отдельного третьего
   канала нет — смысловые такты уровня профиля едут контуром диалога как
   события «результат/такт» с уважением часов тишины; полная картина дня —
   строками дайджеста. Если в v1 лента не нужна вовсе — слово CEO, вычеркну
   одной правкой (помечено как открытая деталь, не блокер G1).
