# INFO-017 — Self-service дистрибуция: продукт тысяч инстансов, два дерева

## Слова CEO (verbatim)

> Стратегический контекст (запиши INFO): Qroky — self-service продукт
> тысяч одиночных инстансов; дистрибуция и автономная работа без
> автора — приоритет №1; каналы пользователя: VSC и Telegram
> равноправно, для нетехнического пользователя Telegram достаточен для
> полного цикла (лента, E1, гейты, результаты — паритет решений с
> сессией).
>
> Дерево A — расширь заказ pilot-kit до distribution kit: install.sh с
> интервью установки (язык ro/ru/en; рабочая папка; проверка/установка
> Claude Code с человеческими подсказками; подписка; Telegram opt-in —
> проводит за руку через BotFather до рабочего токена; телеметрия
> opt-in с показом, что уходит) → идемпотентный, повторный запуск
> безопасен → финал «скажи qroky start». Критерий релиза: чистая
> машина, ≤15 мин, ноль вопросов вне интервью (сухой прогон —
> HUMAN-TASK мне остаётся).
>
> Дерево B (формулируй после A, зависимость — токен из интервью):
> Telegram-голова v1 на инстансе пользователя (его бот-токен, не
> центральный): гейты и E1 кнопками с записью в decisions (паритет с
> сессией — hard-критерий), лента из NARRATIVE.md по уровню детализации
> профиля, утренний дайджест из heartbeat, команда «что в работе» из
> status.yaml. Плюс micro-tier из предыдущего сообщения (уровень 3
> нарратива) — применяй, он входит в тот же контур повествования.

## Interpretation

1. **Форма продукта зафиксирована**: не сервис вокруг автора, а
   self-service — тысячи одиночных инстансов, каждый на машине и
   бот-токене пользователя. Дистрибуция и автономия без автора —
   приоритет №1. Это закрывает риск потери №3 из DIGEST-2026-07-08
   (продукт = автономия) на уровне записи и уточняет RISK-002:
   стандарт и инсталлятор открыты и бесплатны; доверие к исполнению
   (Flow Support, уровень 3 нарратива) — продукт.
2. **Паритет каналов — hard-принцип**: VSC-сессия и Telegram
   равноправны; решение, принятое кнопкой в Telegram, — то же
   конституционное событие (запись в decisions/ по DR5), что и ответ в
   сессии. Для нетехнического пользователя Telegram достаточен для
   ПОЛНОГО цикла: лента (NARRATIVE.md, INFO-015/016), E1, гейты,
   результаты.
3. **Дерево A** = ATOM-100 (distribution-kit-v1): расширение
   071-setup-kit до публичного инсталлятора с интервью. Периметр
   секретов: бот-токен живёт только на машине пользователя. Релиз —
   через сухой прогон CEO на чистой машине (HUMAN-TASK, G2).
4. **Дерево B** = резерв ATOM-110 (telegram-head-v1): формулируется
   ПОСЛЕ доставки A — вход B (рабочий токен и путь его хранения)
   производится интервью A. Hard-критерий B: паритет решений с сессией.
   «Не центральный» — конструктивное решение: у автора нет доступа к
   инстансам, нет бутылочного горлышка и нет позиции оператора
   персональных данных.
5. Уровень 3 нарратива (INFO-016) уже применён (тот же контур
   повествования, что лента дерева B).
