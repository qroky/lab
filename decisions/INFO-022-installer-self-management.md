# INFO-022 — install.sh управляет собой через свой след

## Слова CEO (verbatim)

> Требование к install.sh — переформулировка (от CEO, заменяет прежнее
> про лог): инсталлятор сам управляет собой через свой след. Механика:
> (1) install-state.json — машиночитаемый чекпойнт шагов
> (done/failed/pending + ответы интервью) рядом с человекочитаемым
> логом; (2) каждый шаг — идемпотентная пара «проверь→сделай»:
> перезапуск = прогон проверок сверху, продолжение точно с места
> остановки; повторный запуск на здоровой установке = бесплатный
> health-check («всё стоит, ничего не делал»); (3) при падении —
> самодиагностика: причина по проверке → одна автопопытка из известных
> лекарств (retry сети, ожидание замка, подсказка прав) → не вышло →
> человеку конкретное действие («сделайте Y, запустите снова — продолжу
> отсюда»), НЕ «пришлите файл» (файл — последняя строка для
> безнадёжных); (4) максимум 2 автопопытки на шаг, третье падение —
> всегда человеку; (5) редакция секретов в логе сохраняется
> (grep-критерий пуст). DoD-сценарии обязательные: убить установку на
> середине → перезапуск доходит до конца; запуск на готовой системе →
> чистый health-check; сломанная зависимость → внятная инструкция
> человеку.

## Interpretation

1. Это переформулировка идемпотентности ATOM-100 (прежний H3 «state
   marker») в полную машину состояний: след (install-state.json + лог)
   — не отчёт, а орган управления. Тот же принцип, что file bus
   фреймворка: состояние живёт в артефакте, не в памяти процесса.
2. install-state.json хранит и ответы интервью — перезапуск не
   переспрашивает отвеченное (шесть закрытых пунктов задаются по
   одному разу); это же файл дерево B (ATOM-110) читает как ground
   state инстанса.
3. Пары «проверь→сделай» дают три режима одним механизмом: первичная
   установка, докат после падения, health-check на здоровой системе.
4. Дисциплина падения — лестница: самодиагностика → ≤2 известных
   лекарства → человеку КОНКРЕТНОЕ действие с продолжением с места
   («сделайте Y, запустите снова — продолжу отсюда»); «пришлите файл» —
   последняя строка для безнадёжных, не дефолт. Третье падение шага —
   всегда человеку (никаких бесконечных ретраев).
5. Секреты: редакция в обоих следах (state + лог); проверка —
   негативный grep пуст (расширяет H4).
6. Конверт ATOM-100 поднят ~600k → ~650k (машина состояний + три
   обязательных сценария в dry-run-харнесе); правка внесена
   формулировщиком по INFO-019-практике до запуска, G1-бриф GATE-024
   остаётся действительным — правка исходит от самого CEO.
