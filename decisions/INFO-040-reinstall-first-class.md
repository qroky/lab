  clone в занятую папку упал сырым git fatal)
- **Дата:** 11.07.2026
- **Мораторий INFO-036:** не нарушен — это продуктовый carry-over, не новое
  правило системы.

## Формулировка CEO (дословно)

> Carry-over кита из зачётного прогона, уточнение CEO: переустановка поверх
> существующей папки — легитимный первоклассный сценарий, не ошибка.
> framework/ — пересоздаваемая вендоренная копия (read-only, без
> пользовательских данных по построению), данные живут в qroky/ рядом.
> install-путь обязан обрабатывать: (а) framework/ + данные рядом → диалог
> «переустановка: данные не трогаю, framework пересоздам
> [переустановить/обновить/отмена]» → снос+свежий клон+идемпотентный прогон
> install-state сверху (живое не трогается); (б) framework/ без данных →
> «осиротевший клон, пересоздать?»; (в) чистый случай → обычная установка.
> Финал uninstall ссылается на этот путь («для переустановки просто запусти
> install заново — данные останутся»). Это же — механика восстановления
> после битого self-update. Плюс кандидат SEPARATION-дерева:
> одностраничный bootstrap-скрипт как рекомендуемый вход (clone-логика с
> этой обработкой ДО install.sh, чтобы пользователь не встречал сырой git).
