# INFO-027 — Консистенси-скан нормативного корпуса при freeze

## Слова CEO (verbatim)

> Консистенси-скан (от CEO, в release-чеклист при freeze, парой к
> deletion test): L-tier проход по всему нормативному корпусу
> (конституция, скилл, протоколы, ключевые знания) с задачей найти пары
> норм, предписывающих несовместимое одному актору в одной ситуации.
> Найденное классифицировать, не чинить молча: (а) дефект →
> фикс-кандидат; (б) законная коллизия → явное правило разрешения по
> кодифицируемым принципам старшинства (lex superior = R0.3-направление;
> lex posterior = позднее решение человека бьёт раннее; lex specialis =
> локальное/профильное бьёт общее — кодифицировать секцией в touch №2);
> (в) продуктивное напряжение (конфликт линз by design) →
> документируется как фича. Плюс эмпирический слой: рендер
> «правил-рекордсменов» по логам отклонений/обходов (правило, постоянно
> обходимое с обоснованием, противоречит реальности — кандидат на
> пересмотр). Жёсткое ядро (секреты, push, три человеческих продукта,
> слепота verify) — зона нулевой терпимости: там любой конфликт =
> дефект.

## Interpretation

1. Пара к deletion test (INFO-026): deletion test чистит мёртвое,
   консистенси-скан ловит живое-но-несовместимое. Оба — постоянные шаги
   каждого freeze; скан — строго L-tier (поиск несовместимых пар норм —
   работа класса Verify, MT3-логика).
2. Единица находки — ПАРА норм + актор + ситуация («X обязан A и обязан
   не-A одновременно»), не «мне не нравится формулировка». Классы
   исходов закрыты: (а) дефект → фикс-кандидат в очередь; (б) законная
   коллизия → явное правило разрешения по трём принципам старшинства;
   (в) продуктивное напряжение → документ-фича (by design, как конфликт
   линз SY — прецедент: Роуэн vs соразмерность, INFO-018 §3 —
   разнонаправленная пара, задокументированная как две половины одной
   дисциплины).
3. Принципы старшинства — кандидат 18 touch №2 (секция в конституции):
   lex superior (направление R0.3 — конституция бьёт INPUT, ужесточение
   легально, ослабление нет); lex posterior (позднее записанное решение
   человека бьёт раннее); lex specialis (локальное/профильное бьёт
   общее — направление D1 скилла: правила моста проекта поверх
   RPF-дефолтов).
4. Эмпирический слой: рендер «правил-рекордсменов» из логов отклонений
   и обходов (FP9-переопределения, O5.1-обоснования пропусков,
   deviation-записи RESULT): правило, постоянно обходимое с
   обоснованием, противоречит реальности — кандидат на пересмотр. Это
   рендер уже обязательных записей (дисциплина INFO-016 — не сочинять).
   Живой пример уже в корпусе: INFO-019 (конверт по умолчанию) — обход
   строгого E4, зафиксированный как flagged deviation.
5. Жёсткое ядро — зона нулевой терпимости, класс (б)/(в) там запрещён:
   секреты (S3/H4), push (S4-периметр жеста), три человеческих продукта
   (O8.1), слепота verify (VP2/VP4). Любая пара, ослабляющая ядро, =
   дефект без обсуждения.
6. Первое исполнение — freeze v0.2 (вместе с гигиеной правил INFO-026);
   v0.1 не переоткрывается.
