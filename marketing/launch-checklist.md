# Launch Checklist — Soft Launch (INFO-031, ATOM-120/122)

> Порядок мягкого запуска через тёплую сеть. Каждая рассылка — руками CEO
> (периметр S4/S5); агенты наружу не отправляют ничего. Волны — по 2–3 тёплых
> контакта, между волнами — дни, чтобы фидбек-петля закрылась до следующей.

## 0. Критерий готовности (до волны 1 — оба пункта)

- [ ] Release kit closed (релизный кит закрыт и провязан тегом)
- [ ] ≥1 external user completed install (минимум один внешний человек прошёл установку до конца)

**Жёсткое правило (INFO-024, дословно): «релиз без changelog не публикуется».**

## 1. Порядок шагов

- [ ] **README merge** — ветка `atom-120-launch-package` смержена в `main` qroky/framework после G2 (CEO читает ВСЕ наружные тексты). Исполнитель: CEO.
- [ ] **qroky.com → DNS-редирект на GitHub-репо** — HUMAN-TASK CEO (никакой landing: он вне дерева per INFO-031/GATE-032).
- [ ] **Handoff-сообщение отредактировано CEO** — все три языка (`marketing/handoff/handoff.{en,ru,ro}.md`) — это сырьё под его правку, не готовые тексты (S2).
- [ ] **Волна 1** — CEO лично пересылает сообщение 2–3 тёплым контактам.
- [ ] **Фидбек-петля волны 1 закрыта** — обратная связь идёт через GitHub issues (support-инстанс готовит черновики ответов, отправляет CEO); волна считается закрытой, когда по каждому получателю есть исход: установил / застрял (issue заведён) / молчит N дней.
- [ ] **Волна 2** — через дни после закрытия петли волны 1 (не по календарю — по закрытию петли).
- [ ] **Волна 3** — аналогично, после закрытия петли волны 2.

## 2. Счётчик волн

| Волна № | Дата | Кому (2–3 контакта) | Статус фидбека |
| :------ | :--- | :------------------ | :------------- |
| 1 | — | — | не начата |
| 2 | — | — | не начата |
| 3 | — | — | не начата |

Заполняется CEO (или оркестратором со слов CEO) после каждой отправки; статусы:
`отправлено` → `фидбек идёт` → `петля закрыта (installed: N, issues: M)`.

## 3. Триггер разморозки публичных каналов

HN-/Reddit-драфты и LinkedIn-пост FROZEN (INFO-031, tree-level NOT-DOING, не отказ).
Размораживаются **«после 2–3 волн тёплой раздачи»** — дословный триггер; до его
наступления публичные драфты не пишутся и не публикуются.

## 4. Канал фидбека

GitHub issues репозитория qroky/framework — ЕДИНСТВЕННЫЙ канал обратной связи,
на который указывают README («early access» блок) и handoff-сообщение косвенно
через репо. Ответы: support-инстанс готовит черновик → CEO отправляет.
