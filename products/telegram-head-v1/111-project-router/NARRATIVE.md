# NARRATIVE — ATOM-111 (проектный маршрутизатор)

> Живая лента смысла. Дописывается только в конец (INFO-015/016).

**10.07, старт по «го» (GATE-029).** Твой вопрос «как един бот, если
проектов несколько?» попал в архитектурную дыру v1 — и стал деревом. Цель:
один бот остаётся одним голосом для всех проектов машины — события с
ярлыком «[проект]», решения уходят домой, дайджест один со секциями, мысль
без адреса получает один переспрос кнопками. Жёсткое условие — твой живой
соседний мак обновляется молча: без повторной привязки, без второго «я на
связи»; заодно лечится сегодняшний дефект доката. Исполнитель — тот же
toolsmith (шестой прогон под дисциплиной зрелости), конверт ~250k из ~450k
дерева.

**10.07, развилка «где живёт человек».** Первое настоящее решение: реестр —
единственный новый файл в `~` (ровно так, как разрешил GATE-029), а «дом
человека» — не второй файл рядом, а телеграм-папка ПЕРВОГО проекта из
реестра. Красиво тем, что живой v1-мак после обновления оказывается этим
первым проектом сам собой: ничего не переезжает, ничего не спрашивается.
Второе решение — выключатель совместимости: пока в реестре один проект,
код ведёт себя байт-в-байт как v1. Пятнадцать старых сценариев прошли
нетронутыми — это не удача, это конструкция.

**10.07, харнесс поймал настоящее.** Первый прогон — 19 из 21, и отказ был
не косметикой: на маке путь из харнесса нёс двойной слэш, путь в реестре —
нормализованный, и ожидающий гейт МОЛЧА ИСЧЕЗ из сводного дайджеста. Ровно
тот класс дефекта, ради которого проверки обязаны уметь падать: решение,
которое ждёт человека, не имеет права пропасть из-за косметики пути.
Починено конструкцией, не ослаблением проверки: пути нормализуются при
записи и сравнении, а гейт с «ничейным» происхождением всплывает в секции
первичного проекта — исчезнуть ему больше некуда.

**10.07, закрытие.** Один бот — много проектов: события с ярлыком
«[проект]», решение кнопкой уезжает домой в свой `decisions/`, дайджест
один со секциями (пустой проект — одна строка), «что в работе <имя>»
фильтрует, мысль без адреса получает ровно один переспрос кнопками.
Миграция священна и доказана сценарием: ноль вопросов, ноль повторных
«привет», offset не тронут, старые биты не переигрываются; дефект доката
«токен есть, головы нет» сложен внутрь `--apply-update`. Мёртвый путь в
реестре — честная ⚠-строка, не падение. Голова: 21/21. Кит: 15/15, вопросов
по-прежнему девять. Расход ~175k из ~250k.

**10.07, ночь: принят с нулём открытых находок.** Проверяющий атаковал
маршрутизацию симлинками, двойными слэшами и коллизиями имён — решение
каждый раз уезжало домой; миграцию гонял на v1-состоянии — ноль отправок,
ноль вопросов; «один проект = байт-в-байт v1» проверил буквальным diff'ом
исходящих текстов и подтвердил после фикса склейки. Сам исполнитель на
первом прогоне поймал собственным харнесом настоящий баг (двойной слэш пути
прятал ожидающий гейт из сводного дайджеста) и починил конструкцией.
Экономика дерева: ~273k из ~450k. Тег v0.3 опубликован — твой соседний мак
получит маршрутизатор молчаливой докаткой, а второй проект просто
присоединится к уже подключённому боту. G2 впереди: два живых проекта, гейт
с телефона из каждого, один утренний дайджест.
