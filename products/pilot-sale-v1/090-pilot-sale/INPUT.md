# ATOM-090 — Продажа пилота Startup Moldova (родитель)

## 1. Mission
Строка №1 панели CEO. Условия сделки уже решены (INFO-001); задача — превратить
«согласны в принципе» в назначенный кикофф и двух названных фаундеров.
Это первая задача системы, чей успех определён СНАРУЖИ репозитория; из неё
рождается роль продавца (ROLE-009). Периметр: система готовит, CEO действует.

## 2. Definition of Done
**Hard:**
- H1. ROLE-009 sales-operator существует (карточка роли по образцу roles/).
- H2. Kit существует и полон (§3 ATOM-091).
- H3. Обмен и red lines процитированы из INFO-001 без смягчений и добавок.
- H4. Заходы (RU/RO) — в голосе CEO по style-profile; фаундер-читаемо (S1).
- H5. Сигналы успеха/провала определены бинарно и наблюдаемо.
- H6. Слепой verify: accept.
**Soft:**
- S1. CEO, прочитав kit, может действовать в тот же день без доработки (судья — CEO, G2).

## 3. Success signal (внешний — впервые)
Закрывается НЕ документом: kit принят CEO (G2) → отправлен → сигнал рынка:
дата кикоффа в календаре = продвижение; после кикоффа 2 названных фаундера +
подписанные согласия = продажа состоялась. Сигналы записываются в
workspace/run.log этого атома по мере поступления (long-loop: атом остаётся
открытым до сигнала, форма «долгого атома» — прецедент для touch-очереди).
