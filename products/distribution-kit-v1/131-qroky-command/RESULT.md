## Единственная команда входа (для README и handoff) — отдельная строка

```
bash <(curl -fsSL https://raw.githubusercontent.com/qroky/framework/main/qroky.sh) install
```

git clone ушёл из пользовательского пути во всех трёх README (остался
разделом «альтернатива: клон руками» для air-gapped). После установки —
навсегда и из любого места: `qroky update` / `qroky uninstall`
(+ `qroky details` для полного текста ожидающего обновления).

## DoD 1 — ланчер

`~/.local/bin/qroky`: крошечный, без логики — резолвит `~/.qroky/kit`
(fallback: vendored `framework/qroky.sh` через машинный след — uninstall
работает и офлайн, и без копии кита) и exec'ает qroky.sh. Ставится при
install (через qroky.sh-вход — тем же путём, exec install.sh, ноль
дублирования). PATH-строка — маркер-блоком в профиль шелла ТОЛЬКО если
`~/.local/bin` не в PATH; перед записью маркер ищется во ВСЕХ профилях
(.zshrc, .zprofile, .bashrc, .bash_profile, .profile) — повторный install
не дублирует. Если PATH уже содержит — ничего не пишется. Best-effort:
read-only `~` не убивает установку (честная строка в лог). Честная строка
про НОВЫЙ терминал — в финале (физика INFO-041).

## DoD 2 — периметр I3

Финал (новый `L_FINALE_QROKY_COMMAND` ×3 локали) называет
`~/.local/bin/qroky` и файл профиля с PATH-строкой (когда она была нужна) +
«убирает qroky uninstall». Uninstall: ланчер удаляется ТОЛЬКО с нашим
провенансом (grep INFO-044 — то же правило, что GATE-028 для скилла; чужой
файл на этом пути остаётся с честной строкой), маркер-блок PATH вырезается
awk-диапазоном из любого профиля — чужие строки не тронуты (доказано
сценарием: пользовательская строка в профиле переживает uninstall).
Vendored SKILL.md: I3-исключение дополнено Amended-строкой (INFO-044,
2026-07-11 — третий файл + PATH-строка, удаляются uninstall'ом) + строка
в таблице Provenance; строка GATE-028 сохранена (grep-провенанс жив).
Авторская копия — координатора, не трогал.

## DoD 3 — вход без клона

qroky.sh не имеет ни $0-, ни cwd-предположений — харнес гоняет его через
НАСТОЯЩУЮ process substitution (`bash <(cat qroky.sh) install`) из пустой
нейтральной папки: kit клонируется в `~/.qroky/kit` на свежий релизный
тег, установка идёт оттуда. NOT-DOING соблюдён: только
raw.githubusercontent (никакого qroky.com), никакого /usr/local/bin, sudo,
brew.

## Фикс-раунд (verify ACCEPT r1, находки F1–F4 закрыты)

- **F1 (главная):** qroky.sh диспатчит `enable-telegram | enable-heartbeat |
  enable-backup` — чистый проброс в `install.sh --enable-*` через
  _find_installer (работает и офлайн через vendored installer); ланчер не
  тронут (exec-only, "$@" уже пробрасывал). ВСЕ 48 путезависимых подсказок
  заменены словами: lang ×3 (12 каждый) + README ×3 (4 каждый) →
  «qroky enable-telegram» и т.д.; help qroky.sh называет семью. Харнес:
  сц.22(c3) — реальный проброс `qroky enable-heartbeat` из нейтральной
  папки (heartbeat ON, plist лёг) + sweep-ассерт «install.sh --enable» в
  user-facing исходниках = 0; четыре прежних output-grep'а харнеса
  перенацелены на новые слова (сц.8/9/11/13 — не-вакуумность сохранена).
- **F2:** awk-вырезание PATH-блока поглощает ровно одну ведущую пустую
  строку (однострочный lookbehind) — циклы install/uninstall больше не
  копят пустых строк. Доказано byte-cycle ассертом в сц.22(d2): снапшот
  профиля до добавления blank+block == профиль после uninstall, cmp -s.
- **F3:** сц.22(c2) — второй подряд update (ланчер уже есть, новый тег
  v1.8.0): ноль анонс-строк, тег применён. Тишина теперь доказана прогоном,
  не только кодом.
- **F4:** dry-run.sh делает mkdir -p каталога транскриптов с честной
  строкой на отказе — пустой ATOM_WORKSPACE и распыление в / невозможны.

Харнес после фикс-раунда: **24/24** (сц.22 вырос на 3 ноги: c2, c3,
