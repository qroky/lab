## DoD 1 — три случая точки входа

Реализовано как «reinstall gate» в install.sh (блок ВЫШЕ step_language —
вне awk-диапазона инвентаря вопросов, тот же статус, что у подтверждения
--apply-update: диалог существует только на входе в занятую папку, не в
свежем интервью):

- **(а) framework/ + живые данные** (по построению: живое = install-state /
  CLAUDE.md / файлы в .qroky|.claude|mission|decisions|atoms) → диалог
  `[1 переустановить / 2 обновить / 3 отмена]`; слова всех трёх локалей
  принимаются на любом пути (урок q1); Enter/EOF = отмена — единственный
  безопасный дефолт. «Переустановить» = снос framework/ (включая ЛЮБОЕ
  git-полусостояние: index-запись, .gitmodules-секцию, .git/modules-копию —
  каждое из них по отдельности валило `git submodule add` тем самым сырым
  fatal) + свежий клон + идемпотентный прогон по install-state (ноль
  переспросов, живое не тронуто). «Обновить» = маршрут cmd_apply_update
  (при отсутствии state — честная строка «обновлять нечего» и переспрос).
  «Отмена» = выход без следа (до записи pointer/state).
- **(б) framework/ без данных** → «осиротевший клон, пересоздать?»
  (да/da/yes через is_affirmative); да = снос + чистая установка; нет =
  выход, папка байт-в-байт нетронута.
- **(в) чистое место** → байт-в-байт прежнее поведение (паритет: сценарии
  1/5 зелёные без правок ассертов сути + негативный grep диалоговых строк
  в чистом прогоне, не-вакуумный).
- Вход РЕЗЮМА: завершённая установка → диалог на языке пользователя (сразу
  после восстановления локали); НЕзавершённая (resume после kill) —
  проходит мимо диалога (идемпотентный проход и есть её recovery; сценарий
  2 не тронут). Вход СВЕЖИЙ (перезаклонированный кит, потерян pointer):
  выбранная папка с чужим state УСЫНОВЛЯЕТСЯ (state_load до gate) — прогон
  идёт по записанным ответам; язык этого разговора побеждает сохранённый.
