## Ответ на живой вопрос CEO: почему был EN-финал

Установлено по коду (все закоммиченные версии v0.1 → v0.3 проверены):

- Prompt `--apply-update` **никогда не был EN-hardcode** — с v0.1 это
  `L_UPDATE_ASK_CONFIRM` из lang/*.sh; `is_affirmative` с v0.1 принимает
  yes/y/da/d/да/д/1 на любом пути. Предпосылка мандата (install.sh:887)
  не подтвердилась ни в одной версии.
- Локаль **персистится с v0.1** (`answer_language` в install-state.json) и
  восстанавливается на resume и во всех подкомандах. «Потеря локали между
  шагами» в коде отсутствует.
- Единственные пути к молчаливому EN, оба существовали в коде:
  1. `step_language`: EOF или 5 нераспознанных ответов → `lang="en"`
     **без единой строки предупреждения** (например, ответ «русский»
     прописью не распознавался — только 3/ru);
  2. `${ANSWER_LANGUAGE:-en}` в подкомандах при пустом поле state — тоже
     молча.
- Вывод: EN-финал у CEO = в его install-state.json записано
  `answer_language: en` — либо выбран явно, либо через молчаливый фолбэк
  №1. Проверяется на его машине одной командой:
  `grep answer_language <workdir>/install-state.json`.
- Починено: оба фолбэка теперь печатают честную трёхъязычную строку;
  распознаются написания english/rus/русский/română и т.п. (вопрос не
  переписан — расширен парсинг ответа).
