## Мандат (DoD)

1. **Языковая консистентность всего пути.** Ответ на вопрос 1 (язык)
   управляет КАЖДОЙ видимой пользователю строкой до самого конца:
   - `cmd_apply_update`: prompt «Apply this update now? Type "yes" to
     confirm» — сейчас EN-hardcode (район install.sh:887). Перевести через
     L_*-функции; принимать локализованное подтверждение (да / da / yes —
     на любом из трёх всегда, чтобы «yes» не ломал ru-путь).
   - Провести аудит: grep всех user-visible строк вне lang/*.sh; каждая
     находка — либо L_*, либо логированное обоснование (технический вывод
     harness'а не в счёт).
   - Проверить персистентность локали: q1-ответ сохраняется в
     install-state.json и переиспользуется при `--apply-update`,
     `--enable-telegram`, повторных запусках — без повторного вопроса.
     Разобраться, почему CEO видел EN-финал (выбор EN на q1 — норма;
     потеря локали между шагами — баг).
   - Hello-сообщение бота и все строки telegram-головы, которые видит
     владелец, — на языке q1.
2. **Процедура чистого листа.** Чтобы сквозной тест был честным «с нуля»:
   - `install.sh --uninstall` (или отдельный скрипт в kit): выгружает и
     удаляет launchd-джобы (telegram listener/digest, heartbeat), удаляет
     machine-wide файлы жеста ТОЛЬКО если они наши (маркер-блок в
     ~/.claude/CLAUDE.md, ~/.claude/skills/qroky/ с нашей provenance),
     удаляет ~/.qroky (registry, offsets, state), удаляет install-state.
   - Рабочую папку пользователя (`~/qroky-work` и т.п.) НЕ трогает —
     печатает её путь и говорит: «твои данные здесь, удали сам, если
     хочешь полный ноль». Токен-файл — удаляет с явным предупреждением
     до действия (секрет не читать, только rm). Никаких тихих удалений:
     каждый шаг печатается до выполнения, финал — список сделанного.
   - README (все 3 языка): короткий раздел «Начать с чистого листа».
3. **Харнес.** dry-run сценарии: ru-путь apply-update (prompt на русском,
   «да» принимается, «yes» тоже); локаль переживает apply-update;
   uninstall на машине с полной установкой → повторный install проходит
   как первый (все 9 вопросов, bot hello заново); uninstall на чистой
   машине — вежливый no-op. Ассерты фальсифицируемые (урок r1-F1).
4. **Совместимость.** Существующая установка соседнего мака обновляется
   через `--apply-update` без вопросов (silent migration v0.3 сохраняется).
